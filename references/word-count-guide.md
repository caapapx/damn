# DeepSee 报告字数规范

## 推荐字数范围

### 按场景分类

| 场景 | 字数范围 | 说明 |
|------|---------|------|
| **快速模式** | 800-1200 | 快速决策，Top 3 方案 + 简化 ROI |
| **标准模式** | 2000-3500 | 常规选型，完整分析 |
| **深度模式** | 4000-6000 | 战略级决策，完整证据链 |

### 各章节字数建议

```
总字数: 2000-3500 字（标准模式）

1. 执行摘要          150-250 字
   - 核心结论: 80-120 字
   - 推荐方案: 30-50 字
   - 关键风险: 40-80 字

2. 调研范围          200-300 字
   - 复杂度说明: 50-80 字
   - 数据源列表: 150-220 字

3. 技术方案          600-900 字
   - 每个方案: 200-300 字
   - 优势/劣势: 各 50-100 字

4. 竞品对标          300-500 字
   - 对比表格: 150-250 字
   - 分析说明: 150-250 字

5. ROI 分析          300-400 字
   - TCO 计算: 150-200 字
   - 回本周期: 50-100 字
   - 敏感性分析: 100-100 字

6. 风险评估          250-350 字
   - 风险矩阵: 50-80 字
   - 风险详情: 200-270 字

7. 置信度分析        150-250 字
   - 整体置信度: 30-50 字
   - 分项说明: 120-200 字

8. 行动建议          200-300 字
   - 每项行动: 50-100 字
```

## 为什么这个范围？

### 1. **认知负担平衡**
- **< 800 字**：信息不足，无法支撑决策
- **800-3500 字**：最优范围，用户可在 5-10 分钟内理解
- **> 6000 字**：信息过载，用户容易疲劳

### 2. **Claude 对话流畅性**
- 标准模式 2000-3500 字 = 1-2 个完整回复
- 用户可直接在对话中阅读，无需切换工具
- 便于后续迭代和提问

### 3. **AI 生成效率**
- 快速模式：生成时间 < 30 秒
- 标准模式：生成时间 30-60 秒
- 深度模式：生成时间 60-120 秒

### 4. **实际场景需求**
- **技术团队**：需要详细技术分析（标准模式）
- **管理层**：需要快速结论（快速模式）
- **董事会**：需要完整证据链（深度模式）

## 质量指标

### 不同字数的质量对标

| 指标 | 快速模式 | 标准模式 | 深度模式 |
|------|---------|---------|---------|
| 数据源数量 | 5-8 | 10-15 | 20-25 |
| 方案对比数 | 2-3 | 3-4 | 4-5 |
| 风险识别数 | 2-3 | 3-5 | 5-8 |
| 置信度 | 60-75% | 75-85% | 85-95% |
| 阅读时间 | 3-5 分钟 | 5-10 分钟 | 10-15 分钟 |

## 实施建议

### 1. **自动字数控制**
在 Markdown 生成器中添加字数统计：
```python
def count_words(text):
    return len(text.split())

# 在生成报告时检查
if count_words(report) < 1800:
    print("⚠️ 警告：报告字数过少，可能信息不足")
elif count_words(report) > 4000:
    print("⚠️ 警告：报告字数过多，建议精简")
```

### 2. **模式自适应**
根据数据源数量自动选择模式：
```python
source_count = len(research_scope['sources'])
if source_count <= 8:
    mode = 'fast'      # 800-1200 字
elif source_count <= 15:
    mode = 'standard'  # 2000-3500 字
else:
    mode = 'deep'      # 4000-6000 字
```

### 3. **用户提示**
在生成报告前提示用户：
```
【模式选择】
检测到数据源: 12 个
推荐模式: 标准模式
预期字数: 2000-3500 字
预期阅读时间: 5-10 分钟

继续生成？(y/n)
```

## 常见场景示例

- 字数：1000 字

- 用途：团队内部快速决策
- 示例：选择 Redis vs Memcached

### 场景 2：中等规模技术决策（标准模式）

- 示例：云平台选型（AWS vs GCP vs Azure）

### 场景 3：战略级技术投资（深度模式）
- 字数：5000 字

- 用途：董事会级决策
- 示例：微服务架构迁移方案

## 总结

**推荐标准：**
- **默认**：标准模式 2000-3500 字
- **快速**：快速模式 800-1200 字
- **深度**：深度模式 4000-6000 字

**核心原则：**
- 信息充分但不过载
- 支持决策但不冗长
- 易于理解但不简陋
- 快速生成但不仓促
