# DeepSee 报告格式优化分析

## 问题分析：为什么 HTML 效果不好

### 1. **分发和使用成本高**
- HTML 文件需要浏览器打开，增加用户操作步骤
- 无法直接在 Claude 对话中展示
- 需要额外的文件管理和分享流程
- 移动端体验受限

### 2. **与 Claude 工作流不匹配**
- Claude 的核心输出是文本（Markdown）
- HTML 报告割裂了对话连贯性
- 用户需要在浏览器和 Claude 之间切换
- 无法在对话中快速迭代修改

### 3. **可访问性问题**
- 某些环境无法打开 HTML（企业网络、远程服务器）
- 无法通过文本协议传输（邮件、Slack 等）
- 依赖浏览器渲染，兼容性问题
- 无法版本控制（Git 无法追踪 HTML 变化）

### 4. **数据可视化的真实需求**
- 用户主要需要：**快速理解数据对比**
- ASCII 图表足以满足 90% 的场景
- 精美图表的边际收益递减
- 过度设计反而增加认知负担

---

## 解决方案：Markdown + ASCII 图表

### 优势

✓ **原生集成**：直接在 Claude 对话中展示  
✓ **即时可用**：无需打开外部工具  
✓ **易于修改**：用户可直接复制、编辑、分享  
✓ **版本控制**：完全兼容 Git  
✓ **通用兼容**：任何文本编辑器都能打开  
✓ **快速生成**：无需复杂渲染引擎  

### 实现方式

1. **表格**：Markdown 原生表格（对比矩阵）
2. **柱状图**：ASCII 字符绘制（TCO、成本对比）
3. **热力图**：Unicode 色块 + 数字（方案评分）
4. **折线图**：ASCII 坐标系（置信度演变）
5. **风险矩阵**：四象限 ASCII 图（风险分布）

---

## 实现计划

### 第一步：创建 Markdown 报告生成器
- 输入：JSON 数据（同 HTML 生成器）
- 输出：完整 Markdown 文件
- 包含：所有 8 个章节 + ASCII 图表

### 第二步：ASCII 图表库
- 柱状图：`█` 字符
- 热力图：`█ ▓ ▒ ░` 渐进色
- 折线图：`┌─┐│└┘` 边框 + 数据点
- 风险矩阵：`┼` 十字 + 象限标记

### 第三步：集成到 SKILL.md
- 替换 HTML 方案为 Markdown 方案
- 更新阶段五输出策略
- 保留 HTML 作为可选高级功能

## 建议

**采用 Markdown + ASCII 作为主方案**，原因：
1. 完全符合 Claude 工作流
2. 用户体验更好（无需切换工具）
3. 易于迭代和分享
4. 足以满足技术决策需求
5. 生成和维护成本最低
